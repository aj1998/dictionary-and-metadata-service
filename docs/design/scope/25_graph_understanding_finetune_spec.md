# 25 — Graph Understanding Finetune Spec

Scope context: [`scope/06_advanced_rag_and_finetuning.md`](../../scope/06_advanced_rag_and_finetuning.md) (Graph-understanding model row).
Depends on: [`design/04_data_model_graph.md`](../04_data_model_graph.md) (Neo4j graph + node types), [`design/scope/21_finetune_dataset_export_spec.md`](./21_finetune_dataset_export_spec.md) (dataset row contract), [`design/scope/22_finetune_training_infra_spec.md`](./22_finetune_training_infra_spec.md) (trainer + `finetune_jobs` row), [`design/scope/23_model_serving_registry_spec.md`](./23_model_serving_registry_spec.md) (registry + serve gateway), [`design/scope/24_finetune_eval_harness_spec.md`](./24_finetune_eval_harness_spec.md) (eval scoring).

A finetuned model that answers structural questions about the Jain topic/keyword graph in natural language: parents, children, paths, shared mentions. Trained on synthetic Q/A pairs auto-generated by walking the Neo4j graph, then LoRA-trained on Llama-3.1-8B-Instruct via spec 22, then served by spec 23. v1 goal: replace the brittle hand-tuned ranker in `query-service /graphrag` for the common ancestry / co-mention / path queries.

### Task types and templates

Six question families, each producing both an `instruction` and a verifiable `answer`. Every example is rendered into the standard SFT chat format used by spec 22's `lora_sft` recipe:

```json
{
  "messages": [
    {"role": "system", "content": "You answer questions about the Jain knowledge graph. Be terse and cite topic natural_keys verbatim."},
    {"role": "user",   "content": "<question>"},
    {"role": "assistant", "content": "<answer>"}
  ],
  "meta": {
    "task_family": "parent|children|path|co_mention|ancestry|relation_explain",
    "anchor_natural_keys": ["..."],
    "expected_relations": ["PARENT_OF","RELATED_TO",...]
  }
}
```

**1. parent** — `templates/parent.py`

- Walk: pick a topic `T` with `(T)<-[:PARENT_OF]-(P)`.
- Q: `"What is the parent topic of {T.title_hi} ({T.natural_key})?"`
- A: `"{P.title_hi} ({P.natural_key})"`
- If `T` has no parent: `"{T.title_hi} has no parent in the graph."`

**2. children** — `templates/children.py`

- Walk: pick topic `T` with ≥ 1 child or attached `Keyword`.
- Q: `"List the keywords directly attached to topic {T.natural_key}."`
- A: bullet list of `(keyword.natural_key, keyword.lemma_hi)`, alphabetised by `natural_key`.

**3. path** — `templates/path.py`

- Walk: pick two topics `A`, `B` with `allShortestPaths` of length ≤ `MAX_SUBGRAPH_HOPS`.
- Q: `"What is the shortest relation path from {A.natural_key} to {B.natural_key}?"`
- A: `"{A.natural_key} -[PARENT_OF]-> {C.natural_key} -[RELATED_TO]-> {B.natural_key}"`
- Drop if no path within hop budget.

**4. co_mention** — `templates/co_mention.py`

- Walk: pick two topics `A`, `B` such that ≥ 1 shastra has `Gatha` nodes tagged with both.
- Q: `"Which shastras mention both {A.natural_key} and {B.natural_key}?"`
- A: comma-separated list of `shastra.natural_key`, sorted, capped at 5 with `"…and N more"` suffix when truncated.

**5. ancestry_chain** — `templates/ancestry_chain.py`

- Walk: from `T`, traverse `PARENT_OF` up to the root.
- Q: `"Trace the ancestry of {T.natural_key} up to the root."`
- A: arrow-joined chain `"{root} → ... → {T.natural_key}"`.

**6. relation_explain** — `templates/relation_explain.py`

- Walk: pick a direct edge `(A)-[r]->(B)`.
- Q: `"What is the relation between {A.natural_key} and {B.natural_key}?"`
- A: `"{A.natural_key} {r.type} {B.natural_key}"` (where `r.type ∈ {PARENT_OF, RELATED_TO, MENTIONED_IN, ALIAS_OF}`).

### Determinism contract

Each walker accepts a `seed: int` and uses `random.Random(seed)` only; no global state. `tasks.build_dataset(dataset_name, seed=42)` must produce a byte-identical dataset for the same Neo4j snapshot + seed. This is enforced by `test_walker_determinism.py`.

### Dedupe

`dedupe.py` computes a canonical hash:

```python
def canonical_key(example: dict) -> str:
    q = normalise(example["messages"][1]["content"])   # strip, lowercase, collapse ws
    a = normalise(example["messages"][2]["content"])
    return sha256(f"{q}||{a}".encode()).hexdigest()
```

Drops exact duplicates and near-duplicates (Jaccard on token shingles > 0.92).

### Splitter (no leakage)

`splitter.py` hashes each example's `anchor_natural_keys[0]` modulo 10 → 0–7 train, 8 val, 9 test. This guarantees no topic appears in both train and eval; the eval-time generalisation we care about is across topics, not across phrasings.

### Output

Final dataset is written as JSONL to S3 and a row inserted into `finetune_datasets` (table from spec 21):

```
s3://jinvani-finetune-datasets/graph_understanding/<dataset_name>/
├── train.jsonl
├── val.jsonl
├── test.jsonl
└── manifest.json    {seed, neo4j_snapshot_id, counts, generator_version}
```

`finetune_datasets.task = 'graph_understanding'`; `format = 'sft_chat'`.

## Phase B — LoRA training

### Recipe

`recipes/finetune/graph_understanding/recipe.yaml`:

```yaml
task: graph_understanding
base_model: meta-llama/Llama-3.1-8B-Instruct
recipe: lora_sft
hp_overrides:
  r: 16
  lora_alpha: 32
  lora_dropout: 0.05
  target_modules: ["q_proj","k_proj","v_proj","o_proj"]
  lr: 2.0e-4
  per_device_train_batch_size: 8
  gradient_accumulation_steps: 4
  num_train_epochs: 3
  max_seq_length: 4096
gpu_class: A100-80GB
runtime: modal
```

`hp_defaults.py` in spec 22 already has a row for `(graph_cypher, Qwen-2.5-7B)`; this spec adds a second row for `(graph_understanding, Llama-3.1-8B-Instruct)` with the values above. The trainer (spec 22) is unchanged.

### Trigger

```bash
curl -X POST http://localhost:8001/admin/finetune/jobs \
  -b cookies.txt -H 'content-type: application/json' \
  -d '{"dataset_id":"<id>","base_model":"meta-llama/Llama-3.1-8B-Instruct","recipe":"lora_sft","hp":{},"gpu_class":"A100-80GB"}'
```

On `succeeded`, spec 23's auto-register task creates a `model_registry` row id `graph-understanding-ft-v<n>` in `staging`.

### Prompt format at inference

`recipes/finetune/graph_understanding/prompt.py`:

```python
SYSTEM = ("You answer questions about the Jain knowledge graph. "
          "Be terse and cite topic natural_keys verbatim.")

def format_prompt(question: str, *, context_snippet: str | None = None) -> list[dict]:
    msgs = [{"role": "system", "content": SYSTEM}]
    if context_snippet:
        msgs.append({"role": "system", "content": f"Context:\n{context_snippet}"})
    msgs.append({"role": "user", "content": question})
    return msgs
```

`context_snippet` is optional retrieved-subgraph text injected by `query-service` at runtime; the model was trained with and without it (50/50 split during data gen).

## Phase C — eval + serve

### Eval (uses spec 24 harness)

Hold-out: `test.jsonl` (10% by topic-hash split).

Metrics:

- **Exact match** on the assistant string after whitespace normalisation. Used for `parent`, `relation_explain`, `ancestry_chain` (single-answer tasks).
- **Relation-set F1** for `path` and `co_mention`:
  - Extract `{(src, rel, tgt), ...}` triples from prediction and gold; compute precision/recall/F1 on the triple set.
- **List Jaccard** for `children`: token-set Jaccard over the bullet list items.

Per-task and macro-averaged scores are written to `finetune_jobs.eval_scores` (JSONB) and mirrored to `model_registry.eval_scores`:

```json
{
  "exact_match": {"parent": 0.78, "relation_explain": 0.71, "ancestry_chain": 0.69},
  "relation_f1": {"path": 0.64, "co_mention": 0.58},
  "list_jaccard": {"children": 0.73},
  "macro": 0.685
}
```

Promotion bar: `macro >= 0.60` AND every per-task metric `>= 0.45`. Spec 24's promoter applies this rule when `task='graph_understanding'`.

### Serve (uses spec 23 registry + gateway)

On promotion to `active`:

- Registry row `graph-understanding-ft-v<n>` flips to `status='active'`, `serve_target='vllm'`, `endpoint_url=http://vllm-llama-8b:8000`.
- vLLM container hosts the LoRA adapter:

  ```
  --model meta-llama/Llama-3.1-8B-Instruct
  --enable-lora
  --lora-modules graph-understanding-ft-v1=/checkpoints/graph-understanding-ft-v1/adapter
  ```

- `query-service /graphrag` adds a config flag `USE_GRAPH_FT_MODEL=true` and, when set, replaces the hand-tuned ranker call with:

  ```python
  from jain_kb_common.llm.router import LLMRouter
  router = LLMRouter()
  reply = await router.chat(
      model_id="graph-understanding-ft-v1",
      messages=format_prompt(user_query, context_snippet=retrieved_subgraph_text),
      max_tokens=256, temperature=0.1,
  )
  ```

## Definition of done

- [ ] Synthetic generator produces ≥ 50k deterministic examples against the production Neo4j snapshot.
- [ ] All Phase A unit tests green; end-to-end fixture test green against an embedded Neo4j.
- [ ] LoRA train smoke succeeds on a tiny base; full A100 run completes within cost cap (spec 22).
- [ ] Eval harness writes per-task + macro scores to `finetune_jobs` and `model_registry`.
- [ ] Promotion rule promotes only when bar met; below-bar runs stay in `staging`.
- [ ] Gateway serves the LoRA adapter; `query-service /graphrag` consumes it behind a feature flag and the flag-on path passes its integration test.
- [ ] On a 50-question hand-curated demo set (not from the holdout), the model beats the hand-tuned ranker on macro F1.
