# Phase 04 — `clear_dbs.py --source {all,jainkosh,nj}`

Read [overview](00_overview.md) and complete phases 01–03 first.

## CLI

```
python scripts/clear_dbs.py [--source {all,jainkosh,nj}] [--neo4j-database NAME]
```

Default `--source all` preserves the current behaviour exactly (drop everything).

## Mongo

Split the existing `_MONGO_COLLECTIONS` tuple in
[`scripts/clear_dbs.py`](../../../../scripts/clear_dbs.py) into:

```python
_JK_MONGO = (
    "keyword_definitions", "topic_extracts", "raw_html_snapshots",
    "ocr_pages", "tables",
)
_NJ_MONGO = (
    "gatha_hindi_chhand", "gatha_prakrit", "gatha_sanskrit",
    "gatha_teeka_hindi", "gatha_word_meanings",
    "gatha_teeka_bhaavarth_hindi", "gatha_teeka_bhaavarth_shortfont",
    "gatha_teeka_sanskrit",
    "kalash_bhaavarth_hindi", "kalash_bhaavarth_shortfont",
    "kalash_hindi", "kalash_sanskrit", "kalash_word_meanings",
    "teeka_gatha_mapping",
)
_SHARED_MONGO = ("extract_matches",)  # matcher writes from both feed each other
```

For `--source all`: drop all three groups (current behaviour).
For `--source jainkosh`: drop `_JK_MONGO` + `_SHARED_MONGO`.
For `--source nj`: drop `_NJ_MONGO` + `_SHARED_MONGO`.

(`extract_matches` is regenerated by the matcher; dropping it on either path
is safe.)

## Postgres — per-source delete order

Foreign-key ordering matters. Issue `DELETE ... WHERE $src = ANY(sources)`
in this order (children → parents):

```
keyword_aliases    -- cascades automatically from keywords
topic_candidates   -- only if --source jainkosh (jainkosh-adjacent enrichment)
extract_matches    -- (mongo only; PG has no such table)

ingestion_review_queue WHERE ingestion_run_id IN (SELECT id FROM ingestion_runs WHERE source=$src)

tables           WHERE source = $src
topics           WHERE source = $src
gathas           WHERE $src = ANY(sources)
kalashas         WHERE $src = ANY(sources)
teeka_chapters   WHERE $src = ANY(sources)
publications     WHERE $src = ANY(sources)
teekas           WHERE $src = ANY(sources) AND sources <@ ARRAY[$src]   -- only if exclusively owned
keywords         WHERE $src = ANY(sources) AND sources <@ ARRAY[$src]
shastras         WHERE $src = ANY(sources) AND sources <@ ARRAY[$src]
authors          WHERE $src = ANY(sources) AND sources <@ ARRAY[$src]
books            WHERE $src = ANY(sources) AND sources <@ ARRAY[$src]
pravachans       WHERE $src = ANY(sources) AND sources <@ ARRAY[$src]

ingestion_runs   WHERE source = $src
parser_configs   WHERE source = $src
```

For rows still co-owned (`sources` contains `$src` plus at least one other
source), instead of deleting we **shrink** the array:

```sql
UPDATE shastras
   SET sources = array_remove(sources, $src)
 WHERE $src = ANY(sources) AND cardinality(sources) > 1;
-- repeat for the other shared tables
```

This guarantees the invariant from the overview: two consecutive
`--source jainkosh` then `--source nj` clears bring the DB to the same empty
state that `--source all` produces in one step.

## Neo4j — per-source delete

Two-step pattern per label that owns `sources`:

```cypher
// 1. exclusive nodes: drop
MATCH (n)
WHERE $src IN n.sources AND size(n.sources) = 1
DETACH DELETE n;

// 2. co-owned: shrink
MATCH (n)
WHERE $src IN n.sources AND size(n.sources) > 1
SET n.sources = [s IN n.sources WHERE s <> $src];
```

For `--source all` keep the existing `MATCH (n) DETACH DELETE n` path.

Apply to each label written in phase 03. A small helper in
[`packages/jain_kb_common/jain_kb_common/db/neo4j/clear.py`](../../../../packages/jain_kb_common/jain_kb_common/db/neo4j/clear.py)
encapsulates the two-step.

## Postgres extension / schema rebuild

`--source all` keeps the current "drop & recreate via Base.metadata.create_all"
shortcut. `--source jainkosh|nj` skips the `create_all` rebuild — the schema
already exists; we only DELETE rows.

## Tests — `tests/scripts/test_clear_dbs_per_source.py`

1. **Bootstrap**: ingest both jainkosh goldens (`--shastra-hierarchy`) and at
   least one nj shastra into the test DBs.
2. **Snapshot row counts** per table grouped by `sources` content.
3. Run `python -m scripts.clear_dbs --source jainkosh`. Assert:
   - All exclusively-jainkosh rows / nodes / mongo docs are gone.
   - Co-owned `shastras` rows still exist with `sources = ['nj']`.
   - `gathas`, `kalashas`, `teeka_chapters`, `publications` untouched.
4. Run `python -m scripts.clear_dbs --source nj`. Assert all remaining nj
   rows / nodes / mongo collections are gone — DB is now empty.
5. Re-bootstrap, then run `--source all` and confirm parity with the
   sequential two-clear baseline (row counts identical at 0).

## Manual verification

```bash
python scripts/clear_dbs.py
python scripts/ingest_goldens_apply.py --shastra-hierarchy
python scripts/ingest_nj_apply.py --all

# Clear only jainkosh
python scripts/clear_dbs.py --source jainkosh

psql jain_kb_dev -c "SELECT count(*) FROM gathas;"          # unchanged
psql jain_kb_dev -c "SELECT natural_key, sources FROM shastras;"  # all '{nj}'

# Then clear nj — DB should be empty
python scripts/clear_dbs.py --source nj
psql jain_kb_dev -c "SELECT count(*) FROM shastras;"        # 0
```
